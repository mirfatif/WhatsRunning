    用户界面已被解释

    流程详细信息¶

    1. 进程名称 (命令行)
    2. PID
    3. 用户标识号 / 组标识号
    4. CPU 使用情况
    5. RAM / 交换使用情况
    6. Android 应用软件包计数
    7. 应用任务(活动) 计数
    8. 应用服务计数
    9. 加工年龄

    欲了解更多详细信息，请参阅 screenshots。

    内存信息¶

    详情请参见此处。

    应用服务¶

    1. 应用名称
    2. 服务名称
    3. 服务类型：背景、前景、系统、持久性等。
    4. 绑定服务的客户端应用数量
    5. 服务启动时
    6. 当服务最后激活时
    7. 应用 UID
    8. 应用状态： 正在运行 或 已停止

    活动的应用程序¶

    1. 应用名称
    2. 在应用程序中运行的活动次数(屏幕)和服务
    3. 应用的重要性
    4. 应用程序启动时
    5. 当应用程序最后激活时
    6. 应用 UID
    7. 应用状态： 非常活跃的, 不太活跃的, 或 非活动的
    8. 杀死得分-当设备需要内存时应用程序被杀的几率

    应用状态日志¶

    1. 应用名称
    2. 时间戳 - 当应用程序改变状态时
    3. 新状态 (应用软件包)
    4. 新状态 (应用程序UID)
    5. 应用 UID
    6. 应用进程 PID

    摘要视图¶

    摘要视图显示自应用状态日志启用以来应用已切换到状态的次数。

    1. 应用名称
    2. 新状态
    3. 应用 UID
    4. 计数

    屏幕截图显示，Telegram应用已经切换到41个状态，而Firefox已经打开35次(切换到“前景”状态)。

    闹钟¶

    1. 应用名称
    2. 如果提醒将唤醒设备
    3. 提醒何时会被触发

    预定任务¶

    1. 应用名称
    2. 用户标识符
    3. 作业服务名称
    4. 运行任务前必须满足的条件

    自动启动应用程序¶

    1. 应用名称
    2. 广播接收器名
    3. 用户标识符

    唤醒锁¶

    1. 应用名称
    2. 用户标识符
    3. 唤醒锁名称
    4. 持续时间（锁被保持的时间长度）
    5. 计数 (进行锁的次数)

    唤醒电话¶

    1. 应用名称
    2. 用户标识符
    3. 唤醒设备的闹钟名称
    4. 应用软件包名称
    5. 唤醒计数

    电池哈克¶

    1. 应用名称
    2. 用户标识符
    3. CPU 和网络使用
    4. 应用程序的活动使用时间
    5. 应用程序的前景(高优先级)和背景(低优先级)活动持续时间
    6. 开始的服务总持续时间
    7. 已启动的服务计数
    8. 已启动的服务(仅在下拉菜单中启用时才显示)
    9. 应用名称
    10. 服务名称
    11. 开始服务时间
    12. 服务开始计数

    电池Watcher 服务显示具有电池状态的持续通知：

    1. 电池排流或充电率；温度；估计时间(完全排水或充电)
    2. 设备在电池总耗时的运行时间
    3. 显示时的电池时间；每小时电量下降百分比
    4. 关闭显示时的电池时间；每小时耗电百分比
    5. 设备是 时的电池时间 dozing
    6. 每次耗电量下降的平均耗电时间
    7. 每次耗电量下降的平均网络使用量

    电池状态可以从顶部右侧的屏幕重置。

    WhatsRunning 是什么？

    WhatsRunning 演示您在您的设备上运行了什么，其中包括：

    进程¶

    几乎所有关于本地(Linux)进程的内容：

    • PPID、PID、UID、GID、组、子计数、线程计数
    • SELinux 环境、 Linux 能力 CGroup
    • CPU 用法，RAM 用法，SWAP 使用
    • CPU 优先级，I/O 优先级，OOM杀死得分
    • 命令行、可执行路径、年龄、I/O (存储) 使用情况

    安卓应用¶

    关于框架、系统和用户应用的详细信息：

    • 应用程序名称、包名、重要性、最后活动时间
    • 任务(活动) 名称, 运行状态, 最后活动时间
    • 应用服务名称、 类型、 运行状态、 客户数量、 启动时间、 最后活动时间
    • 初始化服务和框架服务（如适用）

    内存使用情况¶

    • 完全、免费和稀少的内存
    • 活动应用、缓存应用、非应用流程、内核和ZRAM 使用的 RAM
    • 总计交换，应用和非应用进程的使用

    这些详细信息肯定会清除您此后可能对Android的内存管理造成的许多混淆，例如：

    • 为什么您的内存总是完整的，即使您没有使用许多应用？
    • 哪一个应用程序即使在您不使用时也没有被杀死？
    • 缓存的应用程序是什么，它们中的哪些是随后要杀死的 Android 操作系统列表？

    系统服务¶

    Init(原产地)和框架(Java)服务一直在持续运行。

    应用服务¶

    背景(started 和 bound)、 foreground 以及在应用程序进程中运行的持续服务。

    活动的应用程序¶

    应用程序正在运行，无论是在后台还是前台，或处于缓存状态。

    应用状态日志¶

    一个带时间戳的日志，显示应用在活动状态和非活动状态之间切换的情况 状态。

    计划任务¶

    Alarms 可能会反复唤醒设备，并且预定的作业 应用程序在后台运行。

    自动启动应用程序¶

    在设备启动时启动的应用。

    唤醒锁/唤醒电话¶

    那些保持设备不休眠的应用程序。 还有一个计数显示一个应用程序从睡眠中扰动设备的次数。

    电池哈克¶

    (仅适用于 Android 13 的 :disigned_face:)

    在前台和/或后台运行的应用程序的列表比其他应用程序更长。

    电池状态¶

    一个显示当前电池状态，包括电池电量的服务通知， 温度， (解除)电荷速率、估计时间、平均排水速率，包括开关、关闭和戴斯模式。

    更多详细信息请见 screenshots。

    进程状态¶

    服务通知显示运行/死亡进程和系统/用户存活应用的数量。 更多详细信息请参阅Process Watcher。

    另见Github README。
